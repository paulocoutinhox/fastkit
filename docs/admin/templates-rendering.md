# Admin templates and rendering

The admin is **fully server-rendered Jinja** (Django-admin style). A thin jQuery client only enhances
the already-rendered DOM. There is no build step, no SPA, no client-side routing.

## The renderer

`AdminRenderer` (`fastkit_admin/rendering.py`) is a Jinja environment with a `ChoiceLoader` that
searches **consumer override directories before** the package templates. You customize a screen the
Django way — drop a same-named file in your own `templates/` dir.

```python
renderer = AdminRenderer(override_dirs=["myproject/templates"])
```

## The template tree

Templates live under `fastkit_admin/templates/admin/` and are fragmented so you override only the
piece you need:

- `base.html` composes `partials/head.html`, `partials/scripts.html`, and the body.
- `app.html` composes `partials/sidebar.html`, `partials/navbar.html`, and the page: a **page header**
  (breadcrumb) inside `.fk-page-main`, then `.page-body` with `#content`. The **screen title lives in
  the navbar** (`data-testid="screen-title"`), filled from `page_title`.
- `login.html` composes `partials/login_card.html`.
- Repeated markup is generated by macros in `macros.html` (`brand`, `nav_menu`, `asset_link`,
  `asset_script`) so a package change doesn't break an overrider.

## Fill-in partials (inject without copying)

Empty partials are included with `ignore missing`, so you add content by creating them in your override
dir — no need to copy anything:

`_extra_head.html`, `_extra_js.html`, `_pre_body.html`, `_pre_footer.html`, `_post_footer.html`,
`_sidebar_footer.html`, `_navbar_end.html`.

## Screen dispatch

`build_admin_pages_router` server-renders every screen. `dispatch_screen` maps the path
(`resolve_route`) to a screen (dashboard / list / form / detail / report / profile), each
permission-gated and built via `screens.py`. A `?_fragment=table` request renders only the table
partial (what the grid/report AJAX swaps); `?_fragment=form` renders only the bare `<form>` (what the
related-object modal loads).

A `t(key, **params)` callable (bound to the translator + resolved locale, key-fallback) is injected into
every render, so templates translate everything.

See [Override admin templates](../guides/override-templates.md) and [Theme & branding](theme.md).
